Position

The position property specifies the type of positioning method used for an elemnet. The four position values are:

  • Fixed: this will pin something to a spot, and it will stay there as you scroll. (ex: nav bar) Fixed position takes elemnts out of the normal document flow.
  • Static: this is the default setting, it is not affected by top, bottom, left, and right properties. It is always positioned according to the normal flow of the page.
  • Relative: does not take the element out of normal document flow. Still appears within its space/div/container, but allows it to be offset a bit from the top, bottom, right, left. Top, right, bottom, and left properties will move it away from its normal position. Other content will not be adjusted to fit into any gap left by the element. *If postion: relative is set but no other positioning attribute (such as top, bottom, left, right), it will not affect the position at all and treat it as static.
  • Absolute: takes element out of the normal document flow. If a parent element position is set to relative, and the child element is set to absolute, the child element's new (0,0) coordinate will be the upper left hand corner of the parent element instead of the whole page.

The z-index property allows you to override the normal stacking order of the page. The default z-index is 0, and you can set that value to be positive or negative. An element with a greater z-index will stack in front of an element with a lower z-index. If two elemnts have the same z-index, or don't have one assigned, the elemnet positioned last in the HTML will show on top. You must have a position assigned in order to assign a z-index.
